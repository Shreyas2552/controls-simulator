PID Control Law

The PID controller computes:
u(t) = Kp·e + Ki·∫e dt + Kd·(de/dt)

P — reacts to current error (fast, leaves offset)
I — eliminates steady-state error (accumulates history)
D — anticipates future error via rate of change (damps overshoot)

Derivative filter N: practical D-term uses Kd·N·s/(s+N). High N ≈ ideal derivative but noisy; low N = smooth but less effective. Typical: 10–200.

LQR — Linear Quadratic Regulator

Minimises infinite-horizon cost:
J = ∫₀^∞ (x'Qx + u'Ru) dt

Solve Algebraic Riccati Equation:
A'P + PA − PBR⁻¹B'P + Q = 0

Optimal gain: K = R⁻¹B'P, control law: u = −Kx

Kalman Filter (dual of LQR)

Solves: APe + PeA' − PeC'Rn⁻¹CPe + Qn = 0
Observer gain: L = PeC'Rn⁻¹
State estimate: x̂̇ = Ax̂ + Bu + L(y − Cx̂)

Separation Principle (LQG)

LQR + Kalman combined. Design independently:
CL eigs = eig(A−BK) ∪ eig(A−LC)

Q: larger → tighter state tracking (more control effort)
R: larger → cheaper control (slower response)
Qn/Rn: process/measurement noise — tunes observer aggressiveness

MPC — Finite Horizon Optimal Control

At each step k, solve:
min J = Σᵢ Qy·(y_{k+i|k}−r)² + Σⱼ Ru·u_{k+j}²
subject to u_min ≤ u ≤ u_max

Apply u₀ (receding horizon), repeat at k+1.

Prediction Matrices (unconstrained solution)

Y = Φ·x + Ψ·U
U_opt = (Qy·ΨᵀΨ + Ru·I)⁻¹ · Qy·Ψᵀ·(r·1 − Φ·x)

Φ[i] = C·A_d^{i+1} · x (free response)
Ψ[i,j] = C·A_d^{i−j}·B_d (forced response)

Smith Predictor

Removes delay from CL char. equation.
Error: e = r − [y + G_m·u − G_m·e^{−Ls}·u]
With perfect model: e ≈ r − G_m·u (delay-free).
Result: T(s) = C·G/(1+C·G) · e^{−Ls} — delay only on output.

SIMC (Skogestad IMC) PI Tuning

Kp = τ / (K·(λ+L))
Ti = min(τ, 4·(λ+L)) → Ki = Kp/Ti

λ = L: aggressive | λ = τ: moderate | λ = 5τ: conservative

FOPDT Identification from Step Response

K = y_ss / u_step
L ≈ time to reach 5% of y_ss
τ ≈ time to reach 63.2% of y_ss − L

Noisy step responses: average multiple runs or use tangent-line method.

Q: What is Phase Margin and why PM > 45°?

PM = 180° + ∠G(jω_gc). How much extra phase lag before CL goes unstable. PM ≈ 100·ζ (rule of thumb). PM > 45° → ζ > 0.45 → well-damped response without excessive overshoot.

Q: What happens as dead-time L increases?

Delay adds phase lag of −ωL rad (−ωL·180/π degrees). At gain crossover ω_gc: phase shift = −ω_gc·L. PM decreases, eventually to 0 → instability. Rule: PM ≈ 0 when L·ω_gc ≈ PM_orig/57.3.

Q: Controllability & Observability

Controllable: rank([B, AB, A²B, ...]) = n. Every state can be driven to any point in finite time.

Observable: rank([C; CA; CA²; ...]) = n. Every initial state can be determined from outputs.

Required for LQR (controllable) and Kalman (observable).

Q: When use MPC over PID/LQR?

• Hard actuator limits (u_min/u_max) — MPC handles explicitly
• Constraint interactions between channels (MIMO)
• Preview of future reference (known setpoint trajectory)
• Slow processes (enough compute time per step)
Trade-off: MPC needs a model and solves QP online.
